title: SSL/TLS protocol templates
description: Create SSL/TLS templates for certificate validation and TLS security testing

SSL/TLS protocol templates allow you to inspect SSL/TLS certificates, test cipher suites, detect vulnerabilities, and validate TLS configurations.

## Basic SSL request

A simple SSL probe to check certificate information:

```yaml
id: basic-ssl-probe

info:
  name: Basic SSL Certificate Check
  author: pdteam
  severity: info

ssl:
  - address: "{{Host}}:{{Port}}"
    
    matchers:
      - type: dsl
        dsl:
          - "probe_status == true"
```

## SSL request components

### Address

<ParamField path="address" type="string" required>
  Target host and port to probe. Uses `{{Host}}:{{Port}}` for dynamic targeting.
  
  ```yaml
  ssl:
    - address: "{{Host}}:{{Port}}"
  ```
  
  Or specify explicitly:
  ```yaml
  ssl:
    - address: "example.com:443"
  ```
</ParamField>

## TLS version control

### Minimum version

<ParamField path="min_version" type="string">
  Minimum TLS version to use.
  
  **Options**: `sslv3`, `tls10`, `tls11`, `tls12`, `tls13`
  
  ```yaml
  ssl:
    - address: "{{Host}}:{{Port}}"
      min_version: tls12
  ```
</ParamField>

### Maximum version

<ParamField path="max_version" type="string">
  Maximum TLS version to allow.
  
  ```yaml
  ssl:
    - address: "{{Host}}:{{Port}}"
      max_version: tls12
  ```
</ParamField>

### Test for specific TLS versions

```yaml
id: tls10-detection

info:
  name: TLS 1.0 Detection
  author: pdteam
  severity: low
  description: Detects servers still supporting deprecated TLS 1.0

ssl:
  - address: "{{Host}}:{{Port}}"
    min_version: tls10
    max_version: tls10
    
    matchers:
      - type: dsl
        dsl:
          - "probe_status == true"
          - "tls_version == 'tls10'"
        condition: and
```

## Cipher suite testing

### Specific ciphers

<ParamField path="cipher_suites" type="array">
  List of cipher suites to test.
  
  ```yaml
  ssl:
    - address: "{{Host}}:{{Port}}"
      cipher_suites:
        - "TLS_RSA_WITH_RC4_128_SHA"
        - "TLS_RSA_WITH_3DES_EDE_CBC_SHA"
  ```
</ParamField>

### Enumerate ciphers

<ParamField path="tls_cipher_enum" type="boolean" default={false}>
  Enumerate all supported cipher suites.
  
  ```yaml
  ssl:
    - address: "{{Host}}:{{Port}}"
      tls_cipher_enum: true
      tls_cipher_types:
        - insecure
        - weak
  ```
</ParamField>

<ParamField path="tls_cipher_types" type="array">
  Types of ciphers to enumerate.
  
  **Options**: `insecure`, `weak`, `secure`, `all`
  
  ```yaml
  ssl:
    - address: "{{Host}}:{{Port}}"
      tls_cipher_enum: true
      tls_cipher_types:
        - insecure
        - weak
  ```
</ParamField>

## Version enumeration

<ParamField path="tls_version_enum" type="boolean" default={false}>
  Enumerate all supported TLS versions.
  
  ```yaml
  ssl:
    - address: "{{Host}}:{{Port}}"
      tls_version_enum: true
  ```
</ParamField>

## Scan modes

<ParamField path="scan_mode" type="string">
  TLS scanning mode.
  
  **Options**: `auto`, `ctls`, `ztls`, `openssl`
  
  ```yaml
  ssl:
    - address: "{{Host}}:{{Port}}"
      scan_mode: ztls
  ```
</ParamField>

## SSL response data

SSL templates expose extensive certificate and connection data:

### Certificate information

<ResponseField name="subject_cn" type="string">
  Certificate subject common name
  
  ```yaml
  extractors:
    - type: dsl
      dsl:
        - subject_cn
  ```
</ResponseField>

<ResponseField name="subject_org" type="string">
  Certificate subject organization
</ResponseField>

<ResponseField name="subject_dn" type="string">
  Certificate subject distinguished name
</ResponseField>

<ResponseField name="issuer_cn" type="string">
  Certificate issuer common name
</ResponseField>

<ResponseField name="issuer_org" type="string">
  Certificate issuer organization
</ResponseField>

<ResponseField name="issuer_dn" type="string">
  Certificate issuer distinguished name
</ResponseField>

### Validity period

<ResponseField name="not_before" type="timestamp">
  Certificate validity start date
  
  ```yaml
  matchers:
    - type: dsl
      dsl:
        - "not_before > now()"
  ```
</ResponseField>

<ResponseField name="not_after" type="timestamp">
  Certificate expiration date
  
  ```yaml
  matchers:
    - type: dsl
      dsl:
        - "(not_after - now()) < 2592000"  # Expires in 30 days
  ```
</ResponseField>

### Other fields

<ResponseField name="serial" type="string">
  Certificate serial number
</ResponseField>

<ResponseField name="fingerprint_hash" type="string">
  Certificate fingerprint hash
</ResponseField>

<ResponseField name="domains" type="array">
  List of domains in certificate (CN + SANs)
</ResponseField>

<ResponseField name="cipher" type="string">
  Negotiated cipher suite
</ResponseField>

<ResponseField name="tls_version" type="string">
  Negotiated TLS version
</ResponseField>

<ResponseField name="tls_connection" type="string">
  TLS connection type
</ResponseField>

<ResponseField name="probe_status" type="boolean">
  Whether the SSL probe succeeded
</ResponseField>

## Example: Certificate expiration check

```yaml
id: ssl-cert-expiring

info:
  name: SSL Certificate Expiring Soon
  author: pdteam
  severity: medium
  description: Detects SSL certificates expiring within 30 days

ssl:
  - address: "{{Host}}:{{Port}}"
    
    matchers:
      - type: dsl
        dsl:
          - "probe_status == true"
          - "(not_after - now()) < 2592000"  # 30 days in seconds
        condition: and
    
    extractors:
      - type: dsl
        dsl:
          - subject_cn
          - '(not_after - now()) / 86400'  # Days until expiration
```

## Example: Self-signed certificate detection

```yaml
id: ssl-self-signed

info:
  name: Self-Signed SSL Certificate
  author: pdteam
  severity: low
  description: Detects self-signed SSL certificates

ssl:
  - address: "{{Host}}:{{Port}}"

      - type: dsl

          - "subject_dn == issuer_dn"

      - type: dsl
        dsl:
          - subject_cn
          - issuer_cn
```

## Example: Domain validation

```yaml
id: ssl-domain-mismatch

info:
  name: SSL Certificate Domain Mismatch
  author: pdteam
  severity: medium
  description: Detects SSL certificates with domain mismatches

ssl:
  - address: "{{Host}}:{{Port}}"
    
    matchers:
      - type: dsl
        dsl:
          - "probe_status == true"
          - "!contains(domains, Host)"
        condition: and
    
    extractors:
      - type: dsl
        dsl:
          - subject_cn
          - 'join(",", domains)'
```

## Matchers for SSL

### DSL matchers

Most SSL matching is done with DSL:

```yaml
matchers:
  # Check if probe succeeded
  - type: dsl
    dsl:
      - "probe_status == true"
  
  # Check TLS version
  - type: dsl
    dsl:
      - "tls_version == 'tls13'"
  
  # Check cipher
  - type: dsl
    dsl:
      - "contains(cipher, 'AES')"
  
  # Check expiration
  - type: dsl
    dsl:
      - "(not_after - now()) < 86400"  # Less than 1 day
  
  # Check self-signed
  - type: dsl
    dsl:
      - "subject_dn == issuer_dn"
  
  # Check organization
  - type: dsl
    dsl:
      - "contains(tolower(subject_org), 'test')"
```

## Extractors for SSL

```yaml
extractors:
  # Extract subject CN
  - type: dsl
    name: certificate_cn
    dsl:
      - subject_cn
  
  # Extract all domains
  - type: dsl
    name: certificate_domains
    dsl:
      - 'join(",", domains)'
  
  # Extract days until expiration
  - type: dsl
    name: days_until_expiry
    dsl:
      - '(not_after - now()) / 86400'
  
  # Extract issuer
  - type: dsl
    name: issuer
    dsl:
      - issuer_org
  
  # Extract TLS info
  - type: dsl
    name: tls_info
    dsl:
      - 'tls_version + " - " + cipher'
```
